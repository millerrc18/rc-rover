# Next Steps

_Last updated: 2026‑03‑11_

This file contains the prioritised action queue for the `rc-rover` project.  When tasks are completed or priorities change, update this file accordingly.  Tasks should be categorised into **now**, **next**, **later**, and **blocked**.

## Now

These actions should be tackled immediately during the current phase.

- Review and refine the documentation and memory system established in the bootstrap.
- Choose the control architecture for the first rover prototype (e.g. traditional RC transmitter with sensor microcontroller, pure microcontroller with Bluetooth, etc.).
- Research and select candidate chassis platforms and motor configurations suitable for a modular rover.

## Next

These actions come after the "now" tasks are completed.

- Design and build the rolling chassis (stage 0 of the roadmap).
- Implement manual remote control (stage 1), including power distribution and safety features.
- Draft a bill of materials (BOM) for stage 0 and stage 1 hardware.

## Later

These actions are planned but not yet prioritised.

- Add battery and current telemetry sensors.
- Integrate front obstacle sensor and auto‑stop logic.
- Add wheel encoders and speed estimation.
- Integrate an IMU for motion awareness.
- Explore closed‑loop control for speed and heading.

## Blocked

These tasks cannot proceed until the blocking condition is resolved.

- Final naming and branding (blocked until later in the project).
