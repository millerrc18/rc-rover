# Glossary

Definitions of key terms used throughout the project documentation.

- **RC (Radio Control)**: The practice of controlling a vehicle remotely using a wireless transmitter and receiver.  In early phases, this may refer to traditional hobbyist transmitters or Bluetooth controllers.
- **Microcontroller**: A small computing device (e.g. ESP32, STM32) used for real‑time control of motors and sensors.
- **Encoder**: A sensor that measures rotation of a motor shaft or wheel, used to estimate speed or distance travelled.
- **IMU (Inertial Measurement Unit)**: A sensor that measures acceleration and angular velocity in multiple axes, used for motion tracking.
- **Telemetry**: Data collected by the rover (e.g. voltage, current, sensor readings) and transmitted back to the user or logged for analysis.
- **SLAM (Simultaneous Localisation and Mapping)**: The process of building a map of an unknown environment while simultaneously tracking the platform’s position within that map.
- **Assumption**: A statement recorded in the documentation that is believed to be true but has not yet been confirmed.  Assumptions should be clearly marked and revisited later.
