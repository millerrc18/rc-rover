# Start Here

Welcome to the `rc-rover` project.  This project aims to build a modular mobile robotics platform that begins as a simple manually controlled rover and grows into a rich testbed for sensors, control systems and autonomy.

This documentation system is designed to act as a durable memory and operating system for the project.  Everything you need to understand the current state, history, decisions and future plans should be captured here.

## How to use this documentation

1. **Read the high‑level overview** in `../README.md` to understand what the project is and where it is headed.
2. **Review the hand‑off summary** in `HANDOFF.md` for a concise snapshot of the current phase, objectives and next actions.
3. **Consult `PROJECT_STATE.md`** for the canonical description of the current state.
4. **Consult `NEXT_STEPS.md`** to see the prioritised queue of actions.
5. **Use `DECISIONS.md`**, `BUILD_LOG.md` and `LEARNINGS.md` to understand why things were done, how they were done and what was learned.
6. **See `INDEX.md`** for a map of the documentation structure and where to put new content.

## Philosophy

This project will evolve over time.  The documentation should evolve with it.  It is better to capture a summary now than to leave important context in chat.  Use the provided templates in `docs/templates/` to record notes, decisions, experiments and milestones.

## Naming

The project currently uses the working name `rc-rover`.  A final name may be chosen later.  Until then, avoid creating brands or permanent identifiers based on this temporary name.
